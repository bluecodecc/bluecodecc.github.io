日志分析相关命令 2018-08-25

日志分析

通常在项目线上运行时会遇到很多异常，一般都会有日志记录下来，所以可以在日志中寻找出错的地方，但是在如此密集的日志中凭肉眼是很难发现问题的，所以我们可以利用一些简单的命令高效的处理这些问题。

  • 过滤
    grep -V "match_pattern" file_name
  • 查找
    gerp "match_pattern" file_name
    通常简单查找只会显示包含”match_pattern”的一行，并不能显示当时异常发生时候的上下文，此时如果需要显示异常的上下文怎么办?
  • 显示匹配行的指定行数(N)的上下行
    gerp -C N "match_pattern" file_name
  • 结合起来就是
    cat log_file | grep -C N "match_pattern" [| gerp - C N "match_pattern"]
    这样一波下来，如果匹配到的结果还是很多，不利于观察 ,可以重定向到一个文件中
  • 正确姿势
    cat log_file | grep -C N "match_pattern" [| gerp - C N "match_pattern"] >> file_name
