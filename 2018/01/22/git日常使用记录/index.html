git日常使用记录 2018-01-22

在工作中经常遇到这样的情况，这边一个任务还在开发中，那边的测试发现了一个bug需要解决，但是不在同一个分支或者当前还有影响程序运行的bug没有解决。怎么办呢？刚开始可能会有以下几种蠢办法。
1.把这个需求导致的程序修改部分复制出来保存，保持当前代码的正确性。
2.直接提交当前修改部分，重置为远端某个节点（万不能这么做，损人不利己啊！可是开发中很多人多是这么干的，导致别人拉了你的代码后，错误一大堆，根本没法跑起来，而你又不能马上修复，工作就僵在那里了）
以上两种都是不建议的方式，其实可以使用git satsh命令，将当前工作区暂存起来。
git stash pop
可以弹出暂存区最近的工作区
git stash list
可以列出所有被保存起来的工作区（最近的为0）
git stash apply stash@{x}
应用指定的暂存区
git stash clear
删除所有的暂存区
